# Emotion Cards — Phase 4 Launch Assets & Content Plan

**Project:** Emotion Cards  
**Phase:** Phase 4 — Launch  
**Owner:** Gabe 📊  
**Branch Target:** `release/v0.5.0`  
**Status:** Draft for execution

---

## Goal

Ship a lean, coherent launch asset set that sells **Emotion Cards** on its core promise:

- emotional deckbuilding with clear gameplay readability
- character-driven narrative hook
- cozy, reflective tone without feeling passive or low-stakes
- strong enough visual package for Steam, social, creators, and post-launch updates

This plan is intentionally simple. The job is not to make every possible asset — it is to make the **minimum strong set** that supports launch, wishlist conversion, creator coverage, and day-1 store performance.

---

## Core Marketing Message

### Primary Positioning
**A narrative roguelike deckbuilder where your feelings shape your strategy.**

### Supporting Hooks
- Build decks from emotions, memories, and reactions
- Guide Maya through grief, resentment, curiosity, and reconciliation
- Cozy, turn-based play with narrative consequences and replayable runs
- Different emotional paths unlock different story outcomes

### Messaging Rule
Every launch asset should communicate at least **two** of these three pillars:
1. **Deckbuilder strategy**
2. **Emotional narrative**
3. **Distinct visual tone / cozy intimacy**

If an asset only says “story game,” it undersells the strategy. If it only says “deckbuilder,” it loses the differentiation.

## Social Post Types

The goal is coverage, not volume. Reuse the same core creative across formats.

### A) Announcement Posts
Use for:
- launch date reveal
- trailer reveal
- available now
- major update or first patch

Structure:
- one sharp hook line
- one support line about emotion-driven deckbuilding
- CTA: wishlist / watch trailer / play now / read update

### B) Gameplay Feature Posts
Use for:
- emotion card mechanics
- resonance / conflict / transmutation examples
- conversation encounters
- memory dive sequences
- “your deck changes your story” explanation

Format:
- short clip or carousel
- 1 mechanic per post
- plain-language explanation

### C) Character / Narrative Posts
Use for:
- Maya introduction
- sibling conflict setup
- emotional arc framing
- “all endings are valid” tone-setting

Format:
- portrait art or dialogue screenshot
- short narrative blurb
- avoid overexplaining story spoilers

### D) Screenshot Spotlight Posts
Use for:
- showing UI readability
- showing emotional tone
- feeding platforms that reward steady visual posting

Format:
- single screenshot or 3–4 image carousel
- each post tied to one idea: conflict, memory, warmth, confrontation, reflection

### E) Trailer / Clip Distribution Posts
Use for:
- main trailer
- 15-second vertical cutdown
- mechanic micro-clip
- launch day “available now” motion post

### F) Social Proof Posts
Use for:
- creator coverage
- early player reactions
- positive review pull quotes
- milestone beats (wishlists, reviews, patch improvements)

### G) Update / Patch Posts
Use for:
- launch hotfixes
- quality-of-life improvements
- content additions
- community responsiveness

Tone:
- calm
- specific
- trustworthy
- never defensive

## Trailer & Supporting Clips Usage

## Main Launch Trailer
**Purpose:** Steam page conversion, press outreach, trailer reveal, launch day anchor asset.

### Recommended Structure
1. **Hook (0:00–0:08)**
   - strongest line + strongest emotional gameplay visual
   - immediately answer: what makes this different?
2. **Core loop (0:08–0:25)**
   - cards, turns, deckbuilding, emotional states
3. **Narrative hook (0:25–0:40)**
   - Maya, Leo, memory fragments, confrontation stakes
4. **Mechanical differentiation (0:40–0:60)**
   - resonance, conflict, deck-driven choices, multiple paths
5. **Tone + payoff (0:60–0:75)**
   - warmth, grief, reflection, visual identity
6. **CTA end card (final 5–10 seconds)**
   - launch date / available now / platform / wishlist CTA

### Must-Show Trailer Beats
- hand/cards clearly visible in play
- at least one conversation encounter
- at least one memory-related sequence
- at least one deck-management/reward screen
- Maya’s emotional journey hinted, not fully explained
- clear logo and CTA end card

## Supporting Clips
These should be cut from the main capture session, not built from scratch unless necessary.

### Clip Types
- **6–10 second hook clips**
  - fast social teasers
  - strongest visual/mechanical moment
- **10–15 second feature clips**
  - one mechanic, one caption, one CTA
- **15–30 second creator/press support clips**
  - clean gameplay moments for embeds and coverage
- **Vertical cuts**
  - reformatted for short-form platforms
  - use subtitles/burned-in captions if dialogue is central

### Recommended Clip Topics
- “Your feelings shape your deck” clip
- Maya dialogue/confrontation clip
- memory dive clip
- resonance combo clip
- emotional choice / branching result clip
- launch-day montage clip

### Usage Rules
- Trailer = broad story + broad feature sell
- Supporting clips = one idea each
- Recut existing footage before asking for new capture
- Keep clip captions benefit-led, not lore-heavy

## Screenshot / Capsule / Content Checklist

## Screenshot Checklist
Target **8–10** launch-ready screenshots.

### Required Coverage
- [ ] Core combat/encounter UI clearly readable
- [ ] Hand of emotion cards visible and legible
- [ ] Narrative conversation scene
- [ ] Memory dive or reflective scene
- [ ] Reward/deck evolution screen
- [ ] High-tension emotional confrontation moment
- [ ] Warm/cozy visual moment to balance heavier scenes
- [ ] At least 2 screenshots showing different environments or visual moods
- [ ] At least 1 screenshot showing a meaningful card combo/result
- [ ] At least 1 screenshot with Maya or a key character moment

### Screenshot Quality Rules
- no placeholder art
- no debug UI
- no dead/empty compositions
- text should be readable at store thumbnail scale where possible
- avoid 5 screenshots that all look like the same battle from the same angle
- balance gameplay clarity with emotional atmosphere

- [ ] Emotional tone visible immediately
- [ ] Character focus clear enough to be human and memorable
- [ ] Composition still reads in tiny capsule format
- [ ] Color palette distinct from generic dark deckbuilders
- [ ] Doesn’t imply a pure visual novel or pure cozy clicker
- [ ] Matches trailer/store screenshot tone
- [ ] Includes at least one strong focal point, not a crowded collage

- [ ] Genre is stated clearly: narrative roguelike deckbuilder
- [ ] Emotional hook is stated clearly
- [ ] Maya is named consistently if included publicly

## Simple Production Priority Order

This is the minimum sane order so dependencies do not fight each other.

### Priority 1 — Lock Message & Visual Direction
1. finalize positioning/tagline
2. confirm final logo usage and key art direction
3. align on final Steam short description and feature bullets

**Reason:** everything else inherits from this.

### Priority 2 — Capture Build & Core Visual Assets
4. capture final-quality gameplay footage
5. capture/store-select final screenshots
6. produce key art and capsule set

**Reason:** trailer, social clips, store page, and press kit all depend on good source material.

### Priority 3 — Main Conversion Assets
7. edit launch trailer
8. finalize Steam page copy and screenshot order
9. package press kit / fact sheet / B-roll folder

**Reason:** this is the asset layer that actually converts interest into wishlists and purchases.

### Priority 4 — Social Distribution Assets
10. cut trailer into short clips
11. prepare social post variants
12. build launch day and launch week post set

**Reason:** once source assets are approved, distribution becomes straightforward.

### Priority 5 — Post-Launch Support Assets
13. prepare update/news templates
14. prepare review quote template
15. prepare patch messaging framework and reusable art headers

**Reason:** these matter, but only after the launch package exists.

## Lean Execution Recommendation

If time gets tight, protect these first:
- one strong launch trailer
- one polished capsule set
- 8 strong screenshots
- Steam copy that clearly sells the hook
- 4–6 reusable short clips
- one clean press kit/fact sheet

Cut or simplify before you compromise the above.

Lower priority if resources are limited:
- excessive platform-specific copy variations
- too many bespoke motion graphics
- elaborate daily countdown content
- speculative roadmap promises

At launch, Emotion Cards should have:
- a storefront package that clearly communicates **emotional deckbuilding + narrative consequence**

- a trailer + short clip system that supports both Steam and external distribution
- a post-launch messaging template that keeps communication clean after release

That is enough to support a disciplined Phase 4 launch without inflating scope.
