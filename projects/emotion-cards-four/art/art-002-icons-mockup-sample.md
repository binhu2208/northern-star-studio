# Emotion Cards Four — ART-002 Icon Language Sample + Card Mockup

## Purpose
A concise style-exploration slice for **ART-002**, focused on a readable icon language sample and one verbal sample card mockup. This follows the project’s prototype-first goals: fast category recognition, emotional legibility, stream readability, and low-cost iteration.

---

## 1. Icon Language Direction

### Core visual intent
Icons should feel **human, emotionally readable, and system-clean** rather than fantasy-ornate, abstract-techy, or overly cute. The set needs to read instantly at small size in-hand, in the HUD, and on stream.

### Style rules
- Use **simple geometric silhouettes** with one clear idea per icon.
- Favor **rounded interior curves + slightly firm outer shape language** so the system feels empathetic but not soft to the point of vagueness.
- Keep icons readable in **1 color**, with optional 2-tone accent for larger UI states.
- Avoid tiny interior details, facial micro-features, or line noise.
- Build icons on a **consistent stroke weight / corner radius system**.
- Use **solid fill or thick-outline hybrid**, not thin-line iconography.
- Each icon should still read at roughly **20–24 px**.
- Meaning should come from **shape first**, with color as support only.
- Positive vs negative state cues should be obvious through **direction, openness, and tension in the silhouette**.

### Emotional shape bias
- **Trust / supportive / open** = open curves, lifted forms, breathing space
- **Tension / warning / blocked** = pointed compression, sharp closure, stacked pressure
- **Clarity / reveal** = split veil, spark, eye-line, aperture, beam
- **Momentum / movement** = forward tilt, trailing echo, directional flow

## 2. Sample Icon Categories

### A. Card category icons
Use these as the top-level card identifiers.

- **Emotion** — stylized heart/flame hybrid
  - reads as felt internal state, not romance
- **Memory** — folded ribbon-knot / bookmark echo
  - suggests stored context and attachment
- **Reaction** — angled hand/spark burst
  - immediate response, intervention, interruption
- **Shift** — turning prism / rotating arrow arc
  - reframing, redirecting, changing interpretation
- **Breakthrough** — opening star aperture
  - earned release, insight, emotional opening

### B. Encounter stat icons
Use in HUD and compact card support areas.

- **Tension** — compressed zigzag coil
- **Trust** — open linked loop
- **Clarity** — parted cloud / focused beam
- **Momentum** — forward sweep with tail

### C. Response-window / status icons
Small supporting system cues.

- **Open window** — parted gate / open brackets
- **Closed window** — sealed brackets
- **Blocked** — slash-bar over window shape
- **Risk / warning** — tilted triangle pulse
- **Breakthrough ready** — small star aperture with glow ring
- **Collapse danger** — cracked ring / downward split
- **Positive delta** — upward pulse chevron
- **Negative delta** — downward pressure chevron

### D. Keyword / tag marker behavior
Keyword chips should use a **small leading icon + short word label**. Do not rely on icon-only comprehension for secondary mechanics like `misread`, `fragile`, or `defensive`.

## 3. Color + Usage Notes

### Recommended category color families
- **Emotion** — rose / coral
- **Memory** — amber / sepia gold
- **Reaction** — teal / cyan-blue
- **Shift** — violet / indigo
- **Breakthrough** — pale gold / white-gold

### System colors
- **Positive gain** — fresh teal-green pulse
- **Negative pressure** — hot red-magenta pulse
- **Neutral info** — cool gray-blue
- **Blocked / unavailable** — desaturated slate

Color supports recognition, but icons must survive grayscale and disabled states.

## 4. Sample Card Mockup — Verbal Direction

### Sample card choice
**R-001 Guarded Honesty**

This is a strong sample because it sits right in the project’s core promise: honesty that lowers tension, raises clarity, and still carries emotional caution.

### Mockup intent
The card should feel **careful, exposed, and steady** — not explosive, not sentimental. It needs to visually communicate: “I’m telling the truth, but with protection still up.”

### Front-face layout
- **Top band:** Reaction category strip in muted teal with the Reaction icon at upper left.
- **Title area:** “Guarded Honesty” in a clean, slightly condensed serif or humanist display face; enough personality to feel emotional, not fantasy-decorative.
- **Illustration slot:** central vertical panel occupying roughly the upper-middle half of the card.
- **Primary rules box:** short, high-contrast text block beneath the illustration.
- **Conditional rider:** smaller secondary strip with an icon cue showing Memory synergy.
- **Bottom stat rail:** tiny icon row for Clarity up, Tension down, and risk note.

### Illustration direction
The image is **not a full character portrait**. Instead, it is a stylized emotional spot illustration:
- a bust-level silhouette facing three-quarters away,
- one hand near the chest, not fully open,
- a translucent angular veil or pane splitting across the figure,
- a narrow beam of light cutting through the pane,
- soft teal-gray atmosphere with a warmer interior accent at the heart/chest area.

The visual metaphor: the truth is coming through, but filtered through caution.

### Frame treatment
- Reaction frame uses a **cool teal shell** with firmer corners than Emotion cards.
- Outer frame is stable and slightly defensive.
- Interior panel edges soften to keep it human.
- A subtle diagonal accent line suggests interruption / response timing.
- Hover/selected state could brighten the teal and sharpen the clarity beam motif.

### Text styling example
**Primary effect:** `+1 Clarity, -1 Tension`

**Conditional:** `If paired with a Memory card, reveal a safer dialogue option.`

Keep the main effect larger and immediate. Conditional text should feel clearly secondary, not like equal-weight rules clutter.

### Supporting icon usage on the card
- Reaction icon in header
- Clarity-up icon in the effect line
- Tension-down icon in the effect line
- Small Memory icon before the conditional text
- Small risk marker near footer if needed for debug/readability builds

### Overall read at a glance
A player looking for one second should get:
1. **This is a Reaction card**
2. **It helps reveal truth safely**
3. **It improves clarity and reduces tension**
4. **It pairs well with Memory**

That is the right prototype target.

---

## 5. Recommendation for ART-002 Use
This sample direction works best if the rest of ART-002 explores variation in:
- frame ornament density,
- typography pairing warmth vs severity,
- how abstract vs figurative the illustration slot should be,
- how bold the category color separation should be in-hand.

If scope stays tight, keep the icon set and card shell **clean, modular, and slightly understated**. The project wants readability first; over-stylizing this phase would be the wrong move.
